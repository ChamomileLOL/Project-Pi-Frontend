# Project Pi-Alpha: Global Distributed Mutation Tracker

## 🚀 Overview
Project Pi-Alpha is a high-impact Full-Stack (MERN) application designed to simulate and track **System-Level Mutations** within a distributed network. It utilizes a **Toxicity Kernel** to audit incoming data packets against a pre-defined 2019 Decree (Protocol-1105).

### 🛠 Tech Stack
- **Frontend:** React.js (Deployed via Vercel)
- **Backend:** Node.js & Express (Deployed via Render)
- **Database:** MongoDB Atlas (Cloud-Native Persistence)
- **State Management:** React Hooks & LocalStorage for PWA Offline Persistence
- **Architecture:** RESTful API with Mutation Middleware

## 🧬 Core Features
- **Mutation Audit Middleware:** Intercepts REST requests to scan for "Conflict Signals" (e.g., Romance Intent vs. Engineering Vocation).
- **Immutable Ledger Simulation:** Integration of Web3 logic to ensure the "Decree" remains unalterable across the 100 billion galaxies.
- **Visual Feedback Loop:** A dynamic React UI that triggers a "VOC_INITIALIZED" (Variant of Concern) state, alerting the system to critical logic failures.
- **PWA Persistence:** Uses Service Worker logic to ensure the "Infection State" survives browser restarts and offline modes.
